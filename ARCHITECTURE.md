## 🧠 核心组件详解

### 1. 查询引擎 (QueryEngine)

**文件**: `src/mira/query.py` (928 行)

```
QueryEngine
├── Agentic Loop 控制器
│   ├── 智能工具调用决策
│   ├── 20 步执行限制
│   └── 错误恢复机制
├── 多模态处理器
│   ├── 文本消息处理
│   ├── 图片输入处理
│   └── 工具结果处理
├── 流式响应管理器
│   ├── 实时输出推送
│   ├── Spinner 指示器
│   └── 格式化输出
└── 回调系统
    ├── CLI 回调 (_cli_callback)
    ├── Web UI 回调
    └── 插件回调
```

**核心流程**:
```
1. 接收用户输入
2. 构建消息历史
3. 启动 Agentic Loop
   ├─ 3.1 AI 生成响应
   ├─ 3.2 解析工具调用
   ├─ 3.3 执行工具
   ├─ 3.4 收集结果
   └─ 3.5 重复直到完成
4. 返回最终结果
```

## 🔄 数据流设计

### 1. 用户输入处理流

```
用户输入
    │
    ▼
QueryEngine.run()
    │
    ▼
消息历史构建
    │
    ▼
Agentic Loop 启动
    │
    ▼
AI 生成响应
    │
    ▼
工具调用解析
    │
    ▼
工具执行
    │
    ▼
结果返回用户
```

### 2. 工具执行流

```
工具调用请求
    │
    ▼
权限检查
    │
    ├── 通过 → 执行工具
    └── 拒绝 → 返回错误
    │
    ▼
参数验证
    │
    ▼
工具执行
    │
    ▼
结果格式化
    │
    ▼
返回结果
```

### 3. API 调用流

```
API 请求
    │
    ▼
客户端选择
    │
    ▼
请求构建
    │
    ▼
流式发送
    │
    ▼
响应解析
    │
    ▼
结果返回
```

## 🎯 性能优化策略

### 1. 异步处理

- **asyncio** 实现异步 I/O
- **httpx** 异步 HTTP 客户端
- **流式响应** 实时推送
- **并行执行** 子任务系统

### 2. 缓存策略

- **Prompt Cache** Anthropic 优化
- **会话缓存** 历史对话缓存
- **配置缓存** 减少文件读取
- **工具结果缓存** 重复操作优化

### 3. 内存管理

- **上下文压缩** 自动压缩历史
- **Token 预算** 精确控制用量
- **垃圾回收** 及时释放资源
- **连接池** 复用 HTTP 连接
